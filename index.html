What is the Web?

The web is short for "The Internet", and is made up of millions of computers that all talk to each other. Just like a spider's web. When a person wants to go somewhere on the web or searches for something on Google for instance, you start by opening a web browser and typing an address like www.google.com to tell your computer what you want to do. This sends a HTTP request to another computer called a server. The server finds the appropriate HTML document and sends it back to your computer. Your web browser interprets the document and displays the web page on your computer's screen.

What is a Web Page?

A web page is a text document written in a language called HTML. Web browsers reads these documents, interprets the content, and displays it to you.

What is HTLM?

HTML is a collection of documents that lives on these servers, and forms the basis of the web. HTML documents can contain anything - that which you see in your browser - and includes special formatting instructions to show how it looks in your browser. This is called Hypertext Markup Language and is where the name HTML comes from. There is also a special "glue" that holds it all together called a hyperlink, a reference or a web address of where to find the information you're looking for.

How does a HTLM document look?

Just like an English sentence, HTML sentences also have a certain structure, or syntax, in order to make sense. A HTML sentence is called an element, and starts with a tag, may include attributes, followed by the content of the element, and ends with another tag that is the same as the starting tag and signifies the end of the element. For example: "<tag attribute=value> contents </tag>".

Tags are indicated by enclosing them in angled brackets. Attributes have values assigned to them.

These HTML elements are laid out in a special way to indicate the structure of a document. For instance, the element can indicate that one is working with a "HTML" document, with a "head" that identifies the document and includes references to procedures and certain styles in which to display the document, and then a "body" section that is the actual contents of the document - what it is all about.

There are many more rules about things like whitespace, void tags, and different uses for inline and block elements. Fortunately we have an excellent reference in the web itself about HTLM where we can easily look up the syntax. A good place to start is the w3schools.com website.
